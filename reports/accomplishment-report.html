Executive Summary

The NSCC Curriculum Outcome Mapping & Audit Pipeline completed a full institutional sweep of the college's academic inventory, evaluating every course learning outcome across every accredited program — entirely on-premises using locally hosted open-weight AI inference. The pipeline processed 15,459 Learning Outcomes across 3,225 courses and 148 academic programs, producing publication-ready accreditation compliance reports for each, at a direct cloud API cost of $0.00.

This report documents pipeline accomplishments, infrastructure resilience under adverse conditions, the full token workload and equivalent commercial cost, and the strategic value of a local AI architecture for ongoing curriculum quality assurance.

Pipeline Accomplishments

  • End-to-End Extraction & Alignment. Mapped all 15,459 Course Learning Outcomes across 3,225 courses to institutional Program Outcomes across 148 academic programs, producing structured alignment.json files for every program.
  • Bloom's Taxonomy Cognitive Auditing. Every outcome was evaluated for cognitive domain rigor and structural clarity against Bloom's taxonomy. The system identified 1,858 structural deficiencies (12.0%) — outcomes using ambiguous action verbs such as understand, know, or learn that lack measurable evaluation criteria.
  • Automated Accreditation Reporting. Generated 148 publication-ready Word documents (Standards_Compliance_Report.docx) containing compliance breakdowns, cognitive rigor scores, and actionable recommendations for academic chairs and curriculum committees.
  • 100% Execution Reliability. Achieved a 100% JSON parse and structural success rate across all 15,459 evaluations using audit_compliance.py against open-weights Qwen2.5-32B-Instruct-AWQ via local vLLM inference.
  • 27.75 Hours of Unattended GPU Compute. The full two-phase workload ran to completion on the NSCC Truro Campus three-node Proxmox AI cluster (AIProx1/2/3), requiring no manual intervention beyond incident recovery (see below).
Flagged outcomes (12.0%) represent an actionable curriculum improvement backlog. The 1,858 deficient outcomes, surfaced automatically in each program's compliance report, give academic chairs a prioritized list for the next curriculum review cycle — work that would otherwise require manual audit by faculty.

Technical Resilience

The pipeline operated under genuinely adverse infrastructure conditions. The following incidents were encountered and recovered from without data loss:

  • Physical facility power outages disrupted hypervisor PCIe GPU passthrough (WinError 10065) and caused momentary hypervisor OOM freezes during restart boot loops, temporarily taking the vLLM endpoint offline.
  • Idempotent state tracking in run_compliance_batch.py via compliance_audit.json ensured zero-loss recovery. Upon hardware restoration, the batch seamlessly skipped previously audited programs and resumed exactly where processing had stopped — no duplicate processing, no re-evaluation cost.
  • Inference timeout handling in audit_compliance.py caught momentary APITimeoutError events (e.g., on Truck and Transport Repair) and skipped cleanly without crashing multi-hour batch operations.
Key design principle: The batch architecture was built for resilience from the start — idempotent checkpointing meant infrastructure failures translated to minutes of delay rather than hours of lost work.

Strategic Value

Avoiding up to ~$198 USD (~$273 CAD) in direct API fees for this single full-institutional sweep is a concrete near-term saving (equivalent to running the same workload against Anthropic's flagship Claude Fable 5 at current 2026 pricing). The deeper value, however, is structural: local AI inference eliminates per-token cost entirely, turning iterative re-auditing from a budgeted expense into a zero-marginal-cost operation.

Zero Data Egress

Internal curriculum documents, draft pedagogical outcomes, and program evaluations remained entirely within the NSCC private network (10.30.0.x). No institutional IP transmitted to any third party.

Continuous Re-Auditing

As academic chairs refine outcomes based on compliance report recommendations, re-running the full pipeline costs nothing in API fees. Iteration is free.

CI Integration Ready

The idempotent batch architecture supports scheduled runs on a nightly or per-semester basis — enabling Continuous Integration checks for curriculum quality at no additional cost per run.

Scalable to New Programs

Adding a new program requires only dropping source documents into the data/raw/ folder. The pipeline generalizes automatically — no code changes, no additional cloud budget.

Projected annual savings: Across 10–20 iterative re-audit sweeps over an academic year — as outcomes are revised and re-evaluated — local hosting avoids an estimated $1,000–$4,000+ USD ($1,400–$5,500+ CAD) compared to current flagship cloud model pricing (Claude Opus 4.8 at ~$99 USD / ~$136 CAD per sweep; Claude Fable 5 at ~$198 USD / ~$273 CAD per sweep), while maintaining full institutional data custody at zero marginal cost per run.

Recommendations

  • Prioritize the 1,858 flagged outcomes for curriculum committee review in the next academic cycle. Each program's Standards_Compliance_Report.docx contains a prioritized list with specific recommendations.
  • Schedule recurring sweeps — at minimum once per semester — to track outcome quality improvement over time. The pipeline supports this at zero marginal cost.
  • Expand to remaining programs as program outline PDFs and course RTFs become available. The current architecture requires no modification to onboard additional programs.
  • Formalize cluster authentication before broader internal access. The current vLLM endpoint has no authentication layer; network-level isolation is the only access control. Role-based auth should be added prior to sharing access with additional staff.
  • Pin the model version in deployment configuration. LLM outputs are non-deterministic; cluster upgrades that silently change the model version may alter audit results across sweeps. Track model version alongside run timestamps in audit metadata.
